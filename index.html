Command Explorer

kubectl get

Purpose: List resources like pods, services, or deployments.

Example: kubectl get pods -n kube-system

kubectl describe

Purpose: Detailed inspection of a specific resource state and events.

Example: kubectl describe node worker-1

kubectl apply

Purpose: Declarative management of resources via YAML/JSON files.

Example: kubectl apply -f deployment.yaml

kubectl logs

Purpose: Print the logs for a container in a pod.

Example: kubectl logs -f my-pod-xyz
