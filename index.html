System Architecture Components

Control Plane

  • Brain Hub

    kube-apiserver

    Exposes the Kubernetes HTTP API. The front door for all commands.

  • Memory Cell

    etcd

    Consistent, high-availability key-value store for cluster data.

  • Decision Logic

    kube-scheduler

    Assigns newly created Pods to healthy nodes based on resources.

  • Status Monitor

    kube-controller-manager

    Ensures the cluster's current state matches the desired state.

Worker Nodes

  • Agent

    kubelet

    Ensures Pods are running and healthy on each specific node.

  • Network Flow

    kube-proxy

    Maintains network rules for service communication.

  • Engine

    Container Runtime

    The software (like Docker/containerd) responsible for running containers.
